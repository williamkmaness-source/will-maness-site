Overview

Will Maness's personal website, live at willmaness.com. Built to demonstrate technical fluency to VP-of-Product hiring managers at AI-infrastructure and data-tooling companies. The site is a portfolio artifact — it must feel considered, current, and human, not templated.

The site hosts several embedded interactive projects in addition to static content pages: the Boston 311 Equity Tracker, the Vienna Chess Opening Trainer, the Chess Tournament Tracker, the EmberBrief wildfire dashboard, the Vendor Feed preview, and the S&P 500 Technical Analysis dashboard. Each lives under /work/<slug> or at a dedicated top-level route (/ember).

All human-readable copy lives in MDX files under content/, validated at build time against Zod schemas. No CMS — publishing content means creating a file and pushing.

Data Model

All tables live in the neondb database on Neon Postgres. No ORM — raw SQL via @neondatabase/serverless.

Agent Notes

Package manager: Always use pnpm add, never npm install. npm crashes with an arborist null error due to the .pnpm symlink structure.
Vercel project name: The canonical project is website, not will-maness-site. An orphaned duplicate exists on Vercel but has no active GitHub link. Push to main deploys website. Don't reference will-maness-site in deploy commands.
Neon API v1.1 breaking change: The sql(query, params) calling convention was dropped. Use sql.query(query, params) for raw parameterized queries. Tagged template literals (sql\`...\`) still work for simple cases.
DB-backed pages must use force-dynamic: ISR causes build-time prerender failures on Vercel for any page that hits Neon. Add export const dynamic = "force-dynamic" at the top of the route file.
Vercel env vars via REST API: The Vercel CLI plugin blocks vercel env add preview. Use the REST API with the token from ~/Library/Application Support/com.vercel.cli/auth.json.
  • Design tokens: All color, spacing, and typography values come from src/lib/tokens.ts. No raw hex in components.
  • Server components default: Default to server components. Only add "use client" when a component genuinely needs browser state or event listeners.
  • Inline styles bypass Tailwind responsive prefixes: Use className="grid-cols-*" not style={} on layout containers.
  • MDX content validation: Frontmatter is Zod-validated at build time. A missing required field or wrong type breaks the build.
  • Migrations before deploying DB features: Always run any migrate-*.ts script against prod Neon before shipping a new DB-backed page.
  • Cron budget: Vercel Hobby allows max 2 cron jobs at daily cadence. Currently using 1 (ember-ingest). Do not add a third.
  • Featured projects: Only Ember, Chess Tracker, Boston 311, and Vendor Feed are featured on the homepage. Toggle via featured: in content/projects/*.mdx.
  • ANTHROPIC_API_KEY: Present in Vercel Production only — not preview or development. EmberBrief gracefully skips briefing when the key is absent.
  • Run migrations before shipping DB features.
