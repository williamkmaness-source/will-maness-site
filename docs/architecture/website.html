Overview

willmaness.com is a Next.js 16 personal site built to demonstrate technical fluency to VP-of-Product hiring managers at AI-infrastructure and data-tooling companies. The site is the primary artifact proving technical chops; it must feel considered, current, and human — not templated.

All human-readable text lives in MDX files under content/, never in component code. Publishing a new post or project page is creating one file and pushing — no CMS, no dashboard. Frontmatter is validated at build time against Zod schemas so content bugs fail loudly before shipping.

The site hosts several fully-embedded interactive applications: EmberBrief wildfire dashboard, Boston 311 equity tracker, Chess Tournament Tracker, Vienna Opening Trainer, Vendor Intelligence Feed, and SPX Technical Analysis. Each lives at a route under /work/ or /ember.

Agent Notes

⚠️ Use pnpm, not npm. Running npm install crashes with an arborist null error due to the .pnpm symlink structure. Always use pnpm add and pnpm test.
⚠️ DB-backed pages need export const dynamic = 'force-dynamic'. ISR causes build-time prerender failures on Vercel for any route that hits Neon. Add force-dynamic at the top of any new page that queries the DB.
⚠️ Neon v1.1 API changed. Use sql.query(text, params) for raw parameterized queries. The old sql(query, params) form was dropped in v1.1.
⚠️ Vercel project name is "website" (not "will-maness-site"). The will-maness-site project was an orphaned duplicate that was disconnected and deleted. Always target the website project in Vercel CLI commands.
⚠️ Vercel env add preview is broken by a plugin. Bypass with the REST API using the token at ~/Library/Application Support/com.vercel.cli/auth.json.
⚠️ Run migrations before shipping any DB-backed feature. Always run the relevant migrate-*.ts script against production Neon before a new DB-backed page goes live.
💡 Inline styles bypass Tailwind responsive prefixes. Use className="grid-cols-*" instead of style= on layout containers if you need responsive behavior.
💡 ANTHROPIC_API_KEY is Production-only in Vercel. Cron jobs that call Claude must guard against its absence gracefully (log and skip, don't throw).
💡 Design tokens are the single source of truth. No raw hex values in components. All colors/spacing/typography come from src/lib/tokens.ts via Tailwind utilities.
