Overview

A live chess tournament tracker embedded at /work/chess. Auto-discovers the current top Lichess Broadcast, displays round-robin standings and pairings, and lets visitors replay completed games move-by-move on an interactive board. No sign-in, no config, no staleness — 60-second polling keeps data current.

All data is fetched client-side from the public Lichess Broadcasts API. No backend, no database, no server-side state. The entire feature is local to the route — no Zustand, no Context API.

Agent Notes

Rate limit: Lichess enforces rate limits via X-RateLimit-Remaining and X-RateLimit-Reset headers. BroadcastService reads these and switches to 5-minute polling intervals when remaining < 10. Don't bypass or remove this logic.
No backend: This feature has zero server-side code. All API calls go directly from the browser to lichess.org. There is no API route, no Neon table, no cron job.
  • Broadcast selection: The tracker auto-selects the top active broadcast by tier (tier-4 floor, tier-5 preferred). The selection logic lives in BroadcastService — not configurable from the UI in MVP.
  • PGN is lazy: PGN is not pre-fetched for all games. It's fetched when a user clicks a game row in PairingsTable. Cache by FEN/roundId within the session to avoid refetching.
  • chess.js for move parsing: chess.js handles PGN → FEN conversion for each move. ReplayControls manages the move index internally; ChessBoard is purely FEN-driven.
  • Format detection: If the broadcast format is not round-robin (e.g. Swiss), TournamentStandings renders an unsupported-format message. Don't try to render standings for unsupported formats.
