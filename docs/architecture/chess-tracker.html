Chess Tournament Tracker

Live grandmaster tournament tracker — Lichess Broadcasts API, standings, pairings, game replay, move quality, player profiles

Active

Overview

A Chess Tournament Tracker embedded at /work/chess that auto-discovers the current top Lichess Broadcast, displays round-robin standings and pairings, and lets visitors replay completed games move-by-move on an interactive board. No sign-in, no config, no staleness.

The tracker is entirely client-side — no server intermediary. It polls the Lichess Broadcasts API every 60 seconds and backs off to 5-minute intervals when the rate limit gets tight. All state lives in a single useReducer at the top of the component tree.

As of June 2026, the tracker also supports player profile panels (Lichess user data + recent tournament results) and move quality classification via the Lichess Cloud Eval endpoint (Brilliant / Excellent / Good / Mistake / Miss).

Cloud eval is fetched for the FEN before the move; multiPv=5 requested. If the played move is not in the top 5 lines, no quality indicator is shown. If the endpoint returns no result, no indicator shown. Only fires on forward steps (not backward navigation).

Broadcast Selection Logic

Tier filter: ELITE_TIER = 4 (minimum). Prefers tier-5 marquee events when active. Selection priority: ongoing round → last finished round → last round by date. Multi-tournament dropdown shown when multiple elite events are active simultaneously.

Agent Notes

⚠️ No server intermediary — all API calls are client-side. The Lichess API is called directly from the browser. There are no Next.js route handlers for chess data. Do not add server-side caching or database writes for chess tracker state.
⚠️ Rate limit back-off is critical. The Lichess API is rate-limited. When X-RateLimit-Remaining < 10, polling switches from 60s to 5 minutes. Never bypass this check; violating rate limits will get the site blocked.
⚠️ react-chessboard must be dynamically imported with ssr: false. The library is browser-only and heavy. Static import will break the build and add unnecessary weight to the server bundle.
⚠️ chess.js v1 PGN normalization is required. Chess.js fails on multiple consecutive { } comment blocks per move (Lichess includes eval + text + clock as separate blocks). normalizePgn() strips all comments before parsing. Also normalizes 0-0 (zeros) to O-O (letters).
⚠️ GameMoveData.uci is required for move quality. The uci field (e.g., "e2e4") was added in PR #153 to extractGameMoves() via chess.js verbose history (move.from + move.to + promotion). Cloud eval uses UCI format for identifying the played move in PV lines.
💡 Player name → Lichess username mapping. Broadcast PGN headers use Lichess usernames. PlayerService lowercases and strips spaces as a first attempt. A 404 from the user API means the name isn't a valid Lichess ID — show a graceful "Profile unavailable" state, never throw.
💡 BroadcastService NDJSON parsing: The broadcast list endpoint returns one JSON object per line, not a JSON array. Parse with text.split('\n').filter(Boolean).map(JSON.parse). Tournament results endpoint is also NDJSON.
