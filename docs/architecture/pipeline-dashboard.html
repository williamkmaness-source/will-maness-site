Pipeline Dashboard

Real-time health monitoring for the site's data pipelines via Server-Sent Events

Active

Overview

A live dashboard showing the health of the data pipelines powering willmaness.com — currently the Boston 311 nightly pipeline and the Ember wildfire pipeline. The dashboard connects to a Server-Sent Events (SSE) endpoint that emits pipeline status on connect and every 10 seconds thereafter. The browser reconnects automatically on drop.

Each pipeline writes a row to the shared pipeline_runs table in Neon Postgres on every run. The SSE endpoint reads from this table to emit current status. Status cards show last success time, records processed, and a health indicator dot (green/yellow/red/grey).

Agent Notes

⚠️ Every pipeline MUST upsert pipeline_runs on each run. If a pipeline skips this write, the health card shows "Unknown" (grey). This was a historical bug in the 311 pipeline (fixed 2026-05-24). New pipelines added to the system must include this upsert or they'll be invisible to the dashboard.
⚠️ Do NOT use pipeline = 'chess'. The chess tracker does not use a server-side pipeline — it polls Lichess client-side at runtime. Chess rows in pipeline_runs were orphaned and deleted (June 2026, issue #104).
💡 The SSE endpoint emits on connect and every 10 seconds. There is no webhook push when a pipeline run completes — the dashboard polls on a fixed interval. Near-real-time health is a design goal, not strict real-time.
💡 Phantom pipeline failures incident (2026-05-24): A stale branch in the claude_os repo became the default branch, causing all 3 GitHub Actions pipeline workflows to fail daily (they checked out the stale branch). Fixed by resetting the default branch. If pipelines all start failing simultaneously with no code changes, check GitHub Actions default branch configuration first.
