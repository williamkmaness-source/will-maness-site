Vienna Opening Trainer

Interactive chess opening trainer — static theory tree, pure state machine, progressive hints, zero runtime API calls

Complete

Overview

An interactive chess opening trainer embedded at /work/vienna that covers the three principal Vienna Opening lines eight moves deep. The trainer nudges the player when they go off-book and provides progressive hints rather than immediately giving away the answer.

The key design decision: the theory tree is static (pre-computed at build time) rather than fetched from the Lichess Masters database at runtime. Lichess requires OAuth for that endpoint, which adds auth infrastructure for a static artifact that never changes. Five lines are encoded as arrays of UCI move strings, validated by chess.js at build time, and committed as theory.json.

This project is complete — no open issues or planned extensions. It serves as the reference pattern for the chess component architecture: pure state machine, dynamic import for board library, SSR-safe skeleton.

Agent Notes

⚠️ This project is complete — do not add features without explicit request. The trainer covers its intended scope. No open issues.
⚠️ react-chessboard MUST be dynamically imported with ssr: false. The library is browser-only. An SSR-safe skeleton with matching aspect-ratio: 1/1 prevents layout shift while the chunk hydrates.
⚠️ Theory tree is pre-committed — do not re-generate unless lines change. The theory.json file is generated once and committed. Re-running the generator should only happen if the theory lines need to be extended or corrected.
💡 This is the reference pattern for chess components. The Vienna trainer predates the Chess Tracker. When in doubt about how to structure a chess UI component (dynamic import, state machine, sound), treat Vienna as the established pattern.
💡 Two lines in the theory required fixes during generation: a pinned pawn (Bc5 covers f2-g1 diagonal) and illegal castling (king lands on attacked square). chess.js attackers() API was used to detect these. Any future theory expansion should re-validate with chess.js.
