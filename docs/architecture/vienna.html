Overview

An interactive chess opening trainer for the Vienna Game. Presents positions from a curated opening tree and asks the user to find the correct move. Correct moves advance through the theory; incorrect moves show a hint and let the user retry. All state is local — no server, no database, no auth.

The opening theory lives in content/projects/vienna-trainer/theory.json — a hand-curated JSON tree of positions and candidate moves. Updating the theory means editing that file.

Open Issues / Known Gaps

  • Theory tree coverage is manually curated — not exhaustive; only the main Vienna lines are included
  • No spaced-repetition or progress tracking — session resets on page reload
  • No dark mode support (deferred site-wide)

Agent Notes

No backend at all. This component has zero server-side code. Updating theory content = editing theory.json and pushing. No migration, no DB, no API route.
  • Closest existing pattern for chess features: The Vienna trainer was built before the Chess Tournament Tracker. When implementing chess-adjacent features, review both — the Vienna trainer is simpler (no API polling, no PGN replay), while the Chess Tracker adds BroadcastService and the reducer pattern for async state.
  • react-chessboard + chess.js pairing: react-chessboard handles rendering and user input; chess.js handles legality. Keep these responsibilities separate — don't put chess logic in the board component.
  • theory.json schema: Each node in the tree has: FEN, expected move (in SAN or UCI), annotation text, and child nodes. Adding a new line = adding a new branch in the JSON tree.
