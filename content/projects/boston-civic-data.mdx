Boston publishes every 311 service request — every pothole report, every missed trash pickup, every noise complaint — as open data. The dataset goes back to 2011 and updates continuously. What it doesn't do is tell you how fast the city is responding to those requests.

> Can the city of Boston keep up with the volume of requests that are coming in every month? Is the rate of closing these requests meaningfully different across neighborhoods? The data is public, but not clearly summarized - until now.

That's the project. The [Boston 311 Equity Tracker](/work/boston-civic-data) computes a series of visualizations representing how expeditiously the city is closing down 311 tickets. It ranks every neighborhood by the net change in requests and the speed to close those requests with filters by request type to specify certain activities.

## How it works

The city's data lives on the Analyze Boston CKAN platform, split into per-year CSV resources. A GitHub Actions job runs every night, queries the API for a rolling 30-day window of cases, aggregates everything server-side, and upserts the results into a Neon Postgres database. The Next.js route handler reads from Postgres on every request — fast, consistent, and decoupled from CKAN availability. If the database is empty (e.g. on first deploy), it falls back to a live CKAN fetch.

A few deliberate choices in the methodology:

- **Median, not mean.** A single outlier case open for two years drags the mean. Median gives a better read on what a typical resident actually experiences.
- **`reason` field, not `type`.** More stable across years and produces cleaner groupings for filtering.
- **Per-request retry with backoff.** The nightly pipeline retries each CKAN call up to three times with exponential backoff before failing — CKAN can be slow, and a transient hiccup shouldn't abort the run.

Each nightly run accumulates a snapshot row per neighborhood per request type. That history is what will eventually power trend analysis over time.

## What's next

The nightly pipeline is now accumulating snapshots. Once a few weeks of data have built up, the next phase is a trend chart showing how median closure time has moved for the best and worst-performing neighborhoods — turning a point-in-time equity gap into a trajectory. There's also a pipeline health dashboard in progress that surfaces pipeline status and data freshness in real time.
